# Proposal structure reference

## 표준 본문 구조 (한국 R&D 과제 기준)

대부분의 program이 비슷한 골격을 공유한다. 차이는 **분량 제한**과 **강조점**이다.

### 1. 연구개발 목적·필요성

**목표 분량**: 1–2 페이지

**필수 요소**:

- 사회·정책적 필요성(한 단락)
- 시장·산업적 필요성(한 단락 + 정량 데이터)
- 기술적 필요성(한 단락 + 기술 갭)

**Lens 호출 시점**: 시장 규모, 성장률, 기술 갭 수치가 필요할 때

**좋은 패턴**:

> “글로벌 OO 시장은 2026년 X조에서 2030년 Y조로 연평균 Z% 성장이 관측된다. 국내는 핵심 기술 A·B가 해외 의존이다. 본 과제는 A 국산화와 B 확장을 통해 의존도를 낮춘다.” (출처·연도 병기)

**나쁜 패턴**:

- “본 연구는 매우 중요하다”류 자평
- 출처 없는 통계
- 정책 키워드만 나열

### 2. 국내외 기술개발 동향

**목표 분량**: 2–3 페이지

**구조**:

- 글로벌: 선도 기업·연구소 3–5개 비교표(research-synthesis)
- 국내: 경쟁사·정책 흐름
- 우리의 위치: 표 한 칸으로 “차별점 한 문장”

### 3. 연구개발 내용 / 추진방법

**목표 분량**: 3–5 페이지

**구조**:

- 핵심 기술 모듈 3–5개 정의
- 모듈별 정량 지표(정의·측정 방법·목표값)
- 월/분기 일정표(마일스톤·리스크·완화)

### 4. 추진체계

**목표 분량**: 1 페이지

- 주관기관 역할
- 참여기관 역할(있을 때만)
- 의사결정·회의 주기

### 5. 기대효과

**목표 분량**: 1–2 페이지

- 기술: 지표 개선 수치
- 경제: 시장·매출 시나리오(보수/기준)
- 사회·정책: 탄소·안전·일자리 등 공고가 요구하는 축

### 6. 사업비 계획

**목표 분량**: 1 페이지

- 인건비 / 재료비 / 연구활동비 / 간접비 비율(공고 기준 상한 준수)
- 정부출연·기업부담 구분

## 공통 작성 규칙

- 모든 표에 **출처·검증일** 열을 둔다.
- 빈 데이터는 `—`로 두고 추측으로 채우지 않는다.
- 정책 인용은 원문(공고·법령) 기준으로 짧게 인용한다.
