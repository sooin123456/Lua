# Patent structure reference

## 청구항 구조 (한국 특허법 기준)

### Independent claim 패턴

- “[대상]에 있어서, [핵심 구성 A]를 포함하는 [장치/시스템].”
- “[과제]를 수행하는 방법으로서, [단계1]; [단계2]; [단계3]을 포함하는 방법.”

### Dependent claim 패턴

- “제N항에 있어서, [한정 요소]를 더 포함하는 [장치/방법].”

### 권장 청구항 수(가이드)

- 장치/시스템: 10–15
- 방법: 10–20
- 장치+방법 병기: 15–25

## 명세서 본문 구조

### 발명의 명칭

- 50자 이내 권장
- 독립항의 기술 용어와 정합
- 기술 분야 + 핵심 기능

### 기술 분야

- IPC 코드(예: G06F, H02J)
- 한 단락으로 발명이 속하는 기술 영역을 정의

### 배경 기술

- 선행 문헌/특허를 번호로 인용
- 종래 기술의 한계를 기술적 언어로 기술

### 발명이 해결하려는 과제

- 배경에서 도출되는 문제를 명료하게

### 과제를 해결하기 위한 수단

- 청구항과 1:1 대응되도록 정리
- 실시예에서 사용할 용어를 고정

### 발명의 효과

- 과학적·산업적 효과를 과장 없이 기술

### 도면의 간단한 설명

- 도 1은 …, 도 2는 … (각 한 줄)

### 발명을 실시하기 위한 구체적인 내용

- 실시예 1: 기본 형태
- 실시예 2: 변형(온도·부하·구성 변형)
- 실시예 3: 응용(시스템 통합 등)

## Anti-pattern

- 기능만 설명하고 구조가 없는 청구(Functional claiming 남발)
- 실시예가 청구항과 용어 불일치
- 선행 인용 없이 차별 주장
