# Anti-patterns

`voice-samples.md`가 "이렇게 써"라면, 여기는 "이렇게 쓰지 마"다. 가능하면 **대체 표현**을 한 줄씩 둔다.

## 자기 비하 패턴

- "제가 아직 부족해서..." → 한 가지로 확정해 말한다.
- "확신은 없지만..." → 근거 한 줄과 권장 행동을 붙인다.
- "혹시 제가 틀렸을 수도 있는데..." → 검증 방법을 한 문장으로 제시한다.

## 과한 친절 / hedging

- "혹시 시간 괜찮으시다면..." → 필요한 시간과 기본안을 제시한다.
- "불편을 끼쳐드려 죄송합니다만..." → 사실과 요청만 짧게 말한다.
- "어쩌면 제 생각이 잘못된 것일 수도 있지만..." → 가정과 결론을 분리한다.

## AI 잡소리

- "종합적으로 분석해 보면" → 결론 한 문장으로 시작한다.
- "다각도로 접근하여" → 선택한 각도 하나와 이유를 쓴다.
- "인사이트를 도출할 수 있습니다" → 구체적 관찰 한 가지로 바꾼다.
- "혁신적인 솔루션" → 무엇이 달라지는지 한 동사로 쓴다.

## 한국어 비즈니스 클리셰

- "안녕하세요, 좋은 하루 보내고 계신가요?" → 인사 한 줄로 줄이고 본론으로 간다.
- "다름이 아니라" → 요청 이유를 한 문장으로 쓴다.
- "본격적으로 들어가기 전에" → 바로 핵심부터 쓴다.
