# 제안서 양식 패턴 + 시나리오 framework (Round 3.1 확장)

> 새 제안서 들어오면 **3차원으로 분류** — 사업 라인 × 시나리오 × 양식.
> 기존 4종 시나리오는 초기 카탈로그. 사업 라인이 다양해서 확장 가능.

---

### 차원 1: 사업 라인

| 라인 | 영역 | THEKIE 주력 |

## 새 제안서 판별 결정 트리 (확장본)

```
1. 사용자 제공 양식 파일 / 공고문 있나?
   YES → 분석
   NO  → 키워드 기반

2. 사업 라인 판별 (어느 도메인?)
   "에너지 통합관리 / EMS / 모니터링" → BEMS/FEMS/HEMS 중
   "신재생 / 태양광 / 풍력 / 분산자원" → 신재생 관리
   "센서 / 계측기 / IoT / RTU / PV 모듈" → 하드웨어 장치
   "ZEB / 그린리모델링 / 인증" → 인증·정책 대응
   "탄소 / 감축 / DPP / CBAM" → 기후테크
   "창업 / 사업화 / 시드" → 창업·사업화

3. 발주처 판별 (양식 추정)
   조달청 → A
   지자체 → B (시) 또는 E (시나리오 E일 가능성)
   중앙부처 R&D (RS-코드) → C
   인증기관 (KOLAS/GS/ZEB) → D
   중기부/TIPS/창업진흥원 → E
   민간 + 정부 보조 → F
   KGCT → G

4. 컨소 구성 판별 (시나리오)
   신청자 = 단일 기업 → A 또는 G
   주관 = 외부 + 다기관 공동 → D
   주관 = 지자체 → E
   주관 = KGCT → B 또는 G
   주관 = 민간 기업 → F

5. baseline 검색
   past-cases-index.md에서 (사업 라인 + 시나리오 + 양식) 매칭
   매칭 사례 있으면 → 그 사례 사용
   없으면 → 새 패턴으로 등록, 사용자에게 양식·과거 사례 확인 요청
```

## 외부 노출 점검 (모든 양식 공통)

새 제안서 검토 시:

- [ ] "자매 법인" / "동일 CEO" / "그룹 관계" 표현 없는지
- [ ] THEKIE와 KGCT가 같은 컨소에 있을 때 각자 독립 기관처럼 표기됐는지
- [ ] "비용 절감" / "내부 시너지" 표현 없는지
- [ ] KGCT 표기가 일반 용어 ("공동연구개발기관" / "위탁 시험기관" / "협력 기관") 인지
- [ ] 책임자가 양 법인 동일 인물 안 되도록 명시됐는지 (조정훈은 KGCT 측에만 등장)
