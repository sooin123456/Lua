# Automations

## Planning

- [[09_Automations/Notion Workspace Plan|Notion Workspace Plan]]
- [[09_Automations/Slack Command Inbox|Slack Command Inbox]]
 

This folder tracks recurring jobs and cross-tool syncs.

Automations should be documented here before they are enabled. A good automation has an owner, schedule, input, output, rollback path, and failure alert.

## Current Targets

- GitHub Actions for vault checks
- Obsidian to Notion project mirror
- Slack team brief drafts
- weekly review
- inbox triage
- Windows Codex setup
- cleanup backlog
- work provenance ledger

## Rule

Start manual, then semi-automatic, then automatic. Slack and Notion writes should stay draft-only until the team trusts the output.
